CheckinBliss Operational Structure & UI Guidelines
Core Positioning
CheckinBliss is a curated premium marketplace, not an open freelance or general listing platform.
Everything we do is intentional and premium. This fundamentally differentiates us from:

Open marketplaces like Airbnb
Transactional freelance networks

Key rules:

No open listing requests
No open partnership requests
All listings and partnerships are closed and hand-curated

Property Listing Page — Required Changes
Change 1: Remove Individual Property Reviews

Delete: Individual property review ratings and verified tick
Reason: We are a closed, curated, premium marketplace. All apartments are handpicked — we do not rely on user reviews.

Change 2: Simplify Filters & Remove Counts

Delete: “Verified only” filter
Delete: Total number of stays in Lagos (and any similar counters)
Delete: All mass-market style filters that resemble commodity listing platforms

Recommendation (Reference: plumguide.com):

Clean, minimal design
Hidden or very minimal filters
Thin search bar with minimal content
No apartment count displayed (we currently have thin supply)

Change 3: Change Apartment Grid Layout

Delete: Current 3-column apartment grid view

New Recommended Layout:

Left side: Single-column vertical list of properties
Right side: Map view showing apartment locations

Benefits:

Better handles our current low inventory (often 1 page or less)
Solves visual repetition when one property has multiple apartments in the same location
Allows safe mixing of similar-location listings
Creates the illusion of depth and abundance as users scroll (multiple “pages”)

Reference: plumguide.com property listing style
Change 4: Booking Window Communication

Delete: “Booking opens 14+ days ahead” text in checkout

Recommended Approach:

Implement internal logic that:
Returns no availability for any date selection before 14 days
Or greys out unavailable dates in the calendar so users cannot select them


Change 5: User Signup & Verification (Diaspora Focus)
CheckinBliss is built primarily for the diaspora.
Signup Form Updates:

Add Country of Residence field (dropdown)
Dropdown should exclude all African countries
Include only: Europe, USA, Canada, UK, Asia, Australia

Verification Requirements:

Phone number verification via text message (to prevent local bookings)
Consider ID verification
Alternative: Flight ticket verification
